# SNHU-CS330
SNHU CS330

For any software design, assignment, or project the first thing I do is understand the requirements. What is being asked of the program will guide what I build. In this course I focused on building modular and reusable functions to keep the project neat, readable, and functional. My design process started with reviewing the requirements/rubric, then placing the objects in a 3D space. Next adding camera controls allowed me to move about the scene. Following the controls, I added textures and materials to each object. Finally, I added multiple light objects to give the scene a realistic feel. The tactics in this project mirror my approach to any future project, reviewing the design document and requirements, then breaking the project into incremental steps to test and identify problems early. Finally, commenting and organization of my code ensures readability when returning to continue building.  

 

Throughout this project, constant iteration was necessary. When placing the objects in the 3D scene, iteration allowed me to fine-tune the exact placement to replicate the reference picture. Additionally, my lighting went through multiple iterations as I experimented with multiple different types, strengths, and colors to best replicate natural lighting.  

 

This course; Computational Graphics gave me a deeper understanding of graphical api’s and how they translate to creating images. While my focus both educationally and professionally will be outside of 3D graphical design, the broader skillset of writing clean, functional, modular code will translate across any software engineering application. Having the opportunity to dive deeper into C++ specifically, allowed me to better understand classes, pointers, and vectors.
